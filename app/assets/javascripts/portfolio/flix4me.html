Flix4Me See it Live View on GitHub

A fun, interactive app to log your favorite movies -- It’s the Goodreads for movies!

My friends and I love talking about movies. Now that we’re spread across the country, it’s not as easy to do as as when we were in college. I wanted a solution for this akin to how Goodreads does so for books. Enter Flix4Me.

I wanted navigating the app to be lightning fast, so I decided to utilize AngularJS on the front end to make it a single page app. I then created custom Rails API on the back end that seamlessly communicates with the front end. I decided on Postgres for the database for easier future deployment to Heroku. As for styling, I began with Bootstrap 4 then customized it heavily from the stock color palette and options. The app allows users to add, edit, and delete movies to their collection. Currently, movie details must be added manually (title, year, plot, genre, personal rating). Users can comment on movies in their collection and increment the number of times they’ve watched it with a single click.

When implementing the comment feature, the app would not display the newest comment upon creation unless the user navigated away from the page and then returned. Obviously, this is less than ideal. Luckily, this ended up being an easy fix. By using the .push function, the newest comment would be added to the end of the array of all comments. Then, thanks to AngularJS’s ng-repeat directive, the newest comment instantaneously appears at the end of the list of comments on the page. There is a lot in store for Flix4Me. I am planning on adding movie posters to a movie’s details and also incorporating a third party API to allow for a new way to add a movie--just a simple search and click to add. These are just my initial thoughts, but I’m sure there will be more to come.

Utilized an AngularJS front end for a dynamic and responsive front end, allowing for quick searching and filtering features
